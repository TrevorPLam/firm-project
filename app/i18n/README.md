## Configuration Files

### navigation.ts

Exports localized navigation utilities via `createNavigation` from `next-intl/navigation`:

- `Link` â€” Locale-aware link component for internal routing
- `useRouter` â€” Router hook for programmatic locale-aware navigation
- `usePathname` â€” Returns the current pathname without locale prefix
- `redirect` â€” Locale-aware redirect function
- `getPathname` â€” Resolves a localized pathname

Used by components like `LanguageSwitcher`, `Footer`, and page components for locale-aware routing.

### request.ts

Server-side request configuration via `getRequestConfig` from `next-intl/server`:

- Receives locale from the `[locale]` dynamic segment (`requestLocale`)
- Validates locale against supported locales
- Falls back to default locale (`en`) if invalid or missing
- Loads translation messages from `messages/{locale}.json`

Registered in `next.config.ts` via `createNextIntlPlugin("./app/i18n/request.ts")`.

### routing.ts

Central routing configuration via `defineRouting` from `next-intl/routing`:

- Defines supported locales: `en` and `es`
- Sets default locale: `en`
- Consumed by `navigation.ts`, `request.ts`, and the middleware in `proxy.ts`

## Supported Locales

- **English (en)**: Default locale
- **Spanish (es)**: Secondary locale

## Locale Detection

Locale detection is handled by the next-intl middleware in [`proxy.ts`](../../proxy.ts). Priority order:

1. URL path segment (`/en/*` or `/es/*`)
2. Cookie (`NEXT_LOCALE`) â€” persists user's language choice
3. Accept-Language header
4. Default locale (`en`)

## Routing

### URL Structure

- English: `https://example.com/en/about`
- Spanish: `https://example.com/es/about`

### Locale Switching

Language switcher component navigates between locale variants while maintaining the current page.

### Alternate Links

SEO-friendly alternate links for search engines, generated by [`seo-alternates.ts`](../lib/seo-alternates.ts):

```html
<link rel="alternate" hreflang="en" href="https://example.com/en/about" />
<link rel="alternate" hreflang="es" href="https://example.com/es/about" />
```

## Translations

Translation files are located in project root:

- `messages/en.json`: English translations
- `messages/es.json`: Spanish translations

### Usage in Components

Server Components use `getTranslations` from `next-intl/server`:

```typescript
import { getTranslations } from 'next-intl/server';

export async function Page() {
  const t = await getTranslations('HomePage');
  return <h1>{t('hero.title')}</h1>;
}
```

Client Components use `useTranslations` from `next-intl`:

```typescript
import { useTranslations } from 'next-intl';

function Component() {
  const t = useTranslations('Common');
  return <p>{t('loading')}</p>;
}
```

Messages are provided to client components via `NextIntlClientProvider` in `app/[locale]/layout.tsx`.
